2. Переменные Среды (Environment)

Для DBA переменные — это не просто хранилища данных, это способ сказать скрипту, к какому инстансу подключаться.

Ключевые принципы

DB_PORT=5432

Локальная переменная — видна только в текущем скрипте.

export PGDATA="/var/lib/postgresql/data"

Экспорт — делает переменную доступной для всех дочерних процессов (например, для самой утилиты psql).

Значения по умолчанию (Default Values)

# Плохо: скрипт упадет, если переменная не задана
# Хорошо: использование значения по умолчанию
USER_NAME=${DB_USER:-"guest"}
# Если $DB_USER пуста, выберется "guest".
# Это защищает скрипт от пустоты.

3. Потоки данных и Перенаправления

Работа с СУБД — это всегда передача текста. Ты либо отправляешь SQL в базу, либо забираешь логи из неё.

Стандартные дескрипторы

ДескрипторИмяЗначение
stdin (0)Стандартный вводОткуда приходят команды
stdout (1)Стандартный выводРезультат запроса
stderr (2)Стандартные ошибкиСообщения о падении коннекта

Примеры для практики

Слияние потоков

pg_dump my_db > backup.sql 2>&1

Если бэкап упадет, текст ошибки попадет в тот же файл backup.sql — причина будет видна при просмотре файла.

Пайпы (Pipes)

cat script.sql | psql -d mydb

Содержимое файла «пробрасывается» прямо в CLI-интерфейс базы данных.

4. Here-Documents (Инъекция кода)

Это лучший способ писать SQL-запросы прямо внутри Bash-скрипта, сохраняя их читаемость. 

#!/bin/bash
# Переменная внутри Bash
TARGET_SCHEMA="audit_logs"

# Начинаем блок ввода до метки EOF
sqlplus -s / as sysdba <<EOF
SET HEADING OFF;
SET FEEDBACK OFF;
-- Bash подставит значение переменной прямо в SQL:
SELECT count(*) FROM ${TARGET_SCHEMA}.events WHERE status = 'ERROR';
EXIT;
EOF

Лайфхак: Если написать <<'EOF' (в кавычках), Bash не будет подставлять свои переменные внутрь блока — полезно, если в SQL есть свои символы $.

5. Проверка состояния (Exit Codes)

В автоматизации нельзя просто «надеяться», что команда выполнилась. Нужно проверять переменную $?.

0

Команда выполнена успешно

1–255

Произошла ошибка

Пример «Защищенного скрипта»

# Выполняем проверку целостности
mongo --eval "db.adminCommand('checkMetadata')"

if [ $? -ne 0 ]; then
    echo "[$(date)] ERROR: Проверка метаданных MongoDB провалена!" >> /var/log/db_monitor.log
    exit 1
fi

6. Обработка результатов (Command Substitution)

Иногда нужно сохранить результат SQL-запроса в переменную Bash, чтобы принять решение дальше. 

# Записываем текущее количество соединений в переменную
CONN_COUNT=$(psql -t -A -c "SELECT count(*) FROM pg_stat_activity;")

if [ "$CONN_COUNT" -gt 100 ]; then
    echo "Внимание: Высокая нагрузка! Соединений: $CONN_COUNT"
fi

-t -A — флаги PostgreSQL, которые убирают рамки таблиц и заголовки, оставляя только «чистое» число.
