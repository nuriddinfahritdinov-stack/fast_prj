3. Управление инстансом

В Oracle инстанс и база данных — разные сущности. Сначала запускаются процессы (Instance), затем монтируется и открывается база.

Запуск инстанса из Bash

sqlplus / as sysdba <<EOF
STARTUP;
EXIT;
EOF

Проверка статуса (Healthcheck)

Флаг -s (silent) убирает баннеры Oracle и возвращает чистый ответ.

# Проверка: открыта ли база на чтение/запись?
STATUS=$(sqlplus -s / as sysdba <<EOF
SET HEAD OFF FEEDBACK OFF;
SELECT status FROM v\$instance;
EXIT;
EOF
)
echo "Текущий статус инстанса: $STATUS"

Символ \$ экранируется в Bash-скрипте, чтобы он не воспринимался как переменная оболочки.

4. Проверка актуальности бэкапов (RMAN)

Запрос показывает все задачи RMAN за последние 48 часов.

sqlplus -s / as sysdba <<EOF
SET LINESIZE 150 PAGESIZE 20;
COL status FORMAT a10;
COL input_type FORMAT a15;
SELECT input_type, status,
       to_char(start_time, 'YYYY-MM-DD HH24:MI') as start_time,
       time_taken_display as duration
FROM v\$rman_backup_job_details
WHERE start_time > sysdate - 2
ORDER BY start_time DESC;
EXIT;
EOF

5. Мониторинг репликации (Data Guard)

В архитектуре Data Guard необходимо проверять «отставание» (lag) Standby-сервера от Primary.

Проверка задержки Apply Lag

Выполняется на Standby-сервере:

sqlplus -s / as sysdba <<EOF
SET HEAD OFF;
SELECT value FROM v\$dataguard_stats WHERE name = 'apply lag';
EXIT;
EOF

Проверка статуса процессов синхронизации

sqlplus -s / as sysdba <<EOF
COL process FORMAT a10;
COL status FORMAT a12;
SELECT process, status, sequence# FROM v\$managed_standby WHERE process LIKE 'MRP%';
EXIT;
EOF

Если процесс MRP имеет статус APPLYING_LOG — репликация идет в штатном режиме.

6. Анализ активных сессий (Monitoring)

Главный хаб информации о текущей активности — представление V$SESSION.

ТОП-10 сессий по потреблению ресурсов

-- Выполняется в sqlplus или любом GUI
SELECT s.sid, s.serial#, s.username, s.status, s.last_call_et AS duration_sec,
       s.event, s.wait_class, q.sql_text
FROM v$session s
LEFT JOIN v$sql q ON s.sql_id = q.sql_id
WHERE s.status = 'ACTIVE'
  AND s.type != 'BACKGROUND'
ORDER BY s.last_call_et DESC;

last_call_et

Сколько секунд длится текущая операция

event

Чего именно ждет сессия

wait_class

Категория ожидания

Поиск «виновника» блокировок

SELECT sid, serial#, username, blocking_session,
       final_blocking_session, wait_class, event
FROM v$session
WHERE blocking_session IS NOT NULL;

7. Чтение Trace-логов (Диагностика)

Если SQL-запрос выполняется медленно «внутри», стандартных таблиц мониторинга недостаточно — вступают в дело Trace-файлы.

ADR — Automatic Diagnostic Repository

# Узнать путь к диагностическим файлам
sqlplus -s / as sysdba <<EOF
SHOW PARAMETER diagnostic_dest;
EXIT;
EOF

Трейсы текущих сессий находятся в: $DIAG_DEST/diag/rdbms/<db_name>/<instance_name>/trace/

alert_<SID>.log

Главный журнал событий. Ошибки ORA-, перезапуски, проблемы с архивацией.

*.trc

User Trace Files — детальные отчеты о работе конкретной сессии.

Включение трейсирования

-- Включаем сбор детальной статистики для текущей сессии
EXEC DBMS_MONITOR.SESSION_TRACE_ENABLE(waits=>TRUE, binds=>TRUE);
-- После выполнения тяжелого кода выключаем
EXEC DBMS_MONITOR.SESSION_TRACE_DISABLE();
