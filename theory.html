В этой лаборатории вы работаете в эмуляторе терминала Linux, настроенном для обслуживания кластера PostgreSQL 15. Все команды, описанные ниже, ведут себя максимально приближенно к реальной системе Ubuntu/Debian.

Совет для новичков

Используйте Ctrl+L или команду clear для очистки экрана терминала, если вывод стал слишком громоздким.

1. Структура файловой системы

В симуляторе воссоздан стандартный Layout PostgreSQL для Debian/Ubuntu:

/etc/postgresql/15/main/

Конфигурационные файлы: postgresql.conf (главный конфиг), pg_hba.conf (доступ), pg_ident.conf.

/var/lib/postgresql/15/main/

Директория данных (PGDATA). Содержит файлы таблиц, индексов и журналы транзакций (WAL).

/var/log/postgresql/

Серверные логи. Основной файл — postgresql.log. Здесь фиксируются ошибки и события.

/run/postgresql/

Рантайм-файлы: Unix-сокеты (.s.PGSQL.5432) и PID-файл сервера.

3. Продвинутая работа с текстом

Инструменты для фильтрации и обработки данных без открытия тяжелых файлов (напр. логов в десятки ГБ):

Grep — поиск по шаблону

Поддерживает флаги: -i (регистр), -n (номер строки), -v (инверсия), -c (счет), -r (рекурсия).

# Найти все ошибки в логе
grep "ERROR" /var/log/postgresql/postgresql.log

# Найти параметр shared_buffers в конфиге
grep -n "shared_buffers" /etc/postgresql/15/main/postgresql.conf

Sed и Awk — потоковые редакторы

sed удобен для вырезания диапазонов строк, awk — для работы с колонками.

sed -n '10,20p' config.conf

Выведет только строки с 10 по 20.

awk -F= '/port/{print $2}' config.conf

Выведет значение порта (то, что после знака =).

4. Инструменты PostgreSQL

Управление базой данных осуществляется через бинарные файлы и SQL-команды:

Командная строка (Shell)

  • psql — запуск интерактивного терминала БД.
  • psql --version — проверка версии клиента.
  • pg_config --version — информация о сборке сервера.

Внутри PSQL (SQL)

Узнать пути к файлам можно прямо из SQL-сессии:

ЗапросРезультат
SHOW config_file;Путь к postgresql.conf
SHOW data_directory;Путь к PGDATA
SELECT name, setting FROM pg_settings;Все настройки системы

Systemd и Логи ОС

Если сервис упал, используйте systemctl для проверки статуса и journalctl для системного лога (journald).

systemctl status postgresql
journalctl -u postgresql --since "30 minutes ago"

6. Магия Конвейеров (Pipes)

Самая мощная черта Linux — возможность передавать вывод одной программы на вход другой с помощью символа |.

Пример 1: Подсчет количества процессов

ps aux | grep postgres | wc -l

Выведет общее число всех процессов, связанных с PostgreSQL.

Пример 2: Топ-10 самых больших файлов в PGDATA

du -h /var/lib/postgresql | sort -rh | head -n 10

Быстро покажет самые тяжелые таблицы или WAL-логи.

Пример 3: Поиск конкретной ошибки и ее последних вхождений

grep "No space left" pg.log | tail -n 5

Фильтрует сообщения о нехватке места и показывает 5 последних эпизодов.
