3. CLI Reference: Bash + JavaScript

mongosh — это фактически REPL для JavaScript. Ты можешь писать полноценные JS-скрипты и вызывать их из Bash.

Выполнение кода Inline (флаг --eval)

# Простая проверка состояния из Bash
mongosh "$MONGO_URI" --quiet --eval "db.serverStatus().uptime"

Выполнение JS-файлов

# Запуск готового JS-сценария
mongosh "$MONGO_URI" /scripts/maintenance/cleanup_logs.js

The «Inception» Pattern — JS внутри Bash

Генерация JS-кода «на лету» через Here-Documents:

#!/bin/bash
COLLECTION_NAME="users_2024"

# Генерируем JS внутри Bash-контейнера
mongosh "$MONGO_URI" <<EOF
const count = db.${COLLECTION_NAME}.countDocuments({ active: true });
console.log("Active users in ${COLLECTION_NAME}: " + count);
EOF

4. Коварство JavaScript в автоматизации

Хотя JS дает гибкость (циклы, условия, обработка JSON), у него есть обратная сторона:

Ад кавычек

При передаче переменной Bash внутрь JS-строки внутри --eval легко запутаться в ', " и `.

Типизация

В Bash всё — строка. NumberInt(10) и 10.0 (Double) в MongoDB — разные вещи. Ошибка типа может сломать индексы.

Производительность

Правило: максимум логики — в запросе (Aggregation Framework), минимум — в обработке JS-циклом на клиенте.

5. Admin Ops: Однострочники

Быстрые команды для проверки «здоровья» кластера.

Проверка Replica Set

# Узнать, кто сейчас Primary
mongosh --eval "rs.status().members.find(m => m.stateStr === 'PRIMARY').name" --quiet

Управление индексами

# Показать все индексы в коллекции в читаемом виде
mongosh --eval "db.my_collection.getIndexes()" --json

Healthcheck для мониторинга

# Проверка связи и авторизации (вернет 1, если ОК)
mongosh --eval "db.adminCommand('ping').ok" --quiet
